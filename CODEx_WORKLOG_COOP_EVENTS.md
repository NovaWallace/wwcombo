# 协同事件实验模式工作记录

## 目标

- 完成音游反馈窗口重构的收尾：反馈窗口独立、小尺寸、移动模式可见，不抢焦点，不再和主音游框混在一起。
- 修正实验页面侧边栏交互：侧栏 hover 展开时，实验页面保持侧栏收起时的位置和宽度。
- 新增“协同事件”实验子模式，尽量独立于本体录制/练习链路，读取连段谱后在事件块触发音频或图片。

## 本轮原则

- 不回退用户已有改动，不覆盖历史工作树。
- 手工编辑使用 apply_patch。
- 能自主判断的先完成；需要用户确认的放到最后。
- 实验功能参考本体功能模块，但不直接修改录制、练习核心逻辑。

## 未实际完成的手测

- 尚未人工点选窗口验证，因为 `tauri:dev` 是长期运行桌面进程，工具超时后只能确认它进入运行态，无法替代真实屏幕操作。
- 反馈窗口显示、拖动、缩放、点击空白退出移动模式的底层代码已完成，但仍建议在 Tauri 桌面实际窗口里看一次。
- 反馈窗口“单击不抽搐/不缩小”仍建议在桌面实际窗口里点一次确认。
- 反馈窗口“退出移动模式不复原”仍建议在 Tauri 桌面实际窗口中拖动一次再退出确认。
- 协同事件音频播放是否会被浏览器/系统自动播放策略影响，需要实际上传音频后测试。
- 协同事件在练习页真实触发音频/图片仍需桌面端手测：先配置事件块并绑定事件，再到练习页开始，按事件块对应招式确认。
- 已经因旧保存逻辑超限而没有写入的音频绑定无法自动恢复，需要在新版逻辑下重新上传/绑定一次。
- 重复按键导致音频马上暂停的桌面端真实输入仍需手测：按住同一触发键、快速连按触发键、同时开窗口监听和全局监听三种场景都建议试一次。
- 节奏合轴目前使用临时程序化背景和 UI；等用户提供正式素材后，需要替换舞台背景、轨道边框、角色装饰和判定区资产。
- 系统音频捕捉依赖 `getDisplayMedia({ audio: true })` 和用户授权；不同系统/浏览器内核可能只能捕捉屏幕或应用音频，需在 Tauri 桌面端实际确认。
